Version 2.8 of PlusUI brings a plethora of changes and enhancements across the board, and introduces the onboarding UI for first-time visitors. Onboarding: - "FirstVisitExperience" prompts first-time users with a welcome screen and username box - Settings now are now set+saved during FVE to their initial/default values Rolodex: - Further improved new immersive view padding, spacing, and colors - New category bar; click a category to view all of it's cards, or click 'all cards' to view all available cards - New 'info chips'; 3 small chips will show if the current card is parallax, expand, or resize compatible - No longer closes when clicked outside - Cards in the Rolodex now appear in alphabetical order, rather than when they were added - Some cards can now be clicked to instantly expand them, rather than needing the context menu Control panel: - New "Debug code"; gives a ton of useful troubleshooting info in a short code. Click to copy and share. Available on about page. - New “What’s New”; Mirrors changelog found in LATEST GitHub release, scroll to view more. Available on about page. - Touchscreen users will now see slightly tweaked options in control panel - Further improved new rolodex&timedate buttons Cards: - New calculator card; do simple maths straight from the shelf! - New dog/cat pics card; slideshow of adorable dogs/cats, click for new image! - New NASA card; View a random NASA APOD from 1955 onward, click for more details - New Useless Facts card: A random (useless) fact, dedicated to my older brother - Clock card now has a new square style, available in the context menu - Made some changes to the weather card for better performance and retrieval of info UI (Shelf, context menu, etc): - About button now shows contextual information about where you are, and what page you’re on. Rolodex shows current category, control panel shows current page. - The shelf label can now show contextual info about the hovered card - Fixed custom context menu options appearing horizontally in the menu Misc/Meta: - siteDrop is now appended directly to the body rather than to a separate element - Context menu actions have been moved to their own js file
